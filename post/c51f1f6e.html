题目

实现 pow(x, n) ，即计算 x 的 n 次幂函数（即，xn）。不得使用库函数，同时不需要考虑大数问题。

示例 1：
输入：x = 2.00000, n = 10
输出：1024.00000

示例 2：
输入：x = 2.10000, n = 3
输出：9.26100

示例 3：
输入：x = 2.00000, n = -2
输出：0.25000
解释：2-2 = 1/22= 1/4 = 0.25

提示：
-100.0 < x < 100.0
-231 <= n <= 231-1
-104<= xn <= 104

题解

我看到这个题目的时候，以为这是个简单题，直接for循环，不出所料”超出时间限制”，果然没有想象的简单，我看了失败的测试案例：x = 1.00000; n = 2147483647。我一看这必然会出现超时啊，如果for循环的话需要执行n次，太多了。

方法一：递归

思路

我转念一想，执行n次确实太多了，设想210=25 * 25; 25 = 2 * 22 * 22……

an = an/2 * an/2 (n为偶数)
an =a * an/2 * an/2 (n为奇数)

这里有些小技巧如何取分递归中数是奇数还是偶数呢？其实可以利用二进制数的特点判断，如果最后一位是1则为奇数，否则为偶数。位运算的效率比乘除法及求余运算的效率要高很多；还有就是用右移运算符代替了除以2。

在这里再回顾一下如何写一个递归：①确定参数和返回值；②递归终止条件；③递归内部逻辑。写之前我在想递归的话是直接使用所给的函数直接递归，还是重新写一个函数递归调用；后来发现需要重新写一个函数，因为函数的参数类型需要变化，n为负数的时候最小值取正的时候会超过数据类型int的范围，导致报错。
