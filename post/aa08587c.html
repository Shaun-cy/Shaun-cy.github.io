剑指offer04题解

发表于2022-09-15|更新于2022-09-16

题目

​ 在一个 n * m 的二维数组中，每一行都按照从左到右递增的顺序排序，每一列都按照从上到下递增的顺序排序。请完成一个高效的函数，输入这样的一个二维数组和一个整数，判断数组中是否含有该整数。

现有矩阵 matrix 如下：
[
[1, 4, 7, 11, 15],
[2, 5, 8, 12, 19],
[3, 6, 9, 16, 22],
[10, 13, 14, 17, 24],
[18, 21, 23, 26, 30]
]
给定 target = 5，返回 true。
给定 target = 20，返回 false。

限制：
0 <= n <= 1000
0 <= m <= 1000

题解

方法一：找规律

思路

​ 分析一下数组的特点，尝试选取一个数从第一个数出发判断。若等于则输出，然而大于当前位置的数，便出现两种选择：向下或向右。由于要查找的数字相对于当前位置有可能在两个区域中出现，并且两个区域还有重叠，此时从这个角度似乎没有出路，但是我们不妨换个思路从数组的其他角出发。

比如从数组的右上角：假定目标数为20，从数组的右上角数字15开始进行判断：（设 行下标为row;列下标为col）
20>15：因为15为当前行最大的数，所以20不可能出现在当前行 —> row++ 数组下标来到了数字：19
20>19：同理row++ 数组下标来到了数字：22
20<22：因为22往下的数是越来越大，所以20不会出现当前列 —> col– 数组下标来到了数字：16
20>16：row++ 数组下标来到了数字：17
20>17：row++ 数组下标来到了数字：26
20<26：col– 数组下标来到了数字：23
20<23：col– 数组下标来到了数字：21
20>21：超出数组边界 说明数组中无当前数

方法二：二分搜索法

思路

​ 由题可知，单看某一行或者某一列，它都是有序数组，要在其中查找一个便想到二分搜索法！（有序数组查找问题—>二分搜索法）
对于这个题二分搜索有三个思路：

  1. 依次对行（列）进行二分搜索，没有搜索到则跳转下一行（列）
  2. 先将数组拥有的“行数”和“列数”进行比较；若行数小则对行进行二分搜索，若列数小则对列进行二分搜索
  3. 同时对行、列进行搜索，同时推进。

​ 以上三个思路供参考，至于具体各个思路的时间、空间复杂度面我还不是很了解。

总结

​ 有序数组的查找问题，最直接的思路便是二分法！本题是二维数组，另一个思路便是不妨用一个具体的数去探索数组中存在的规律，进而尝试求解。
​ 本题一定程度上考察了答者的分析问题的能力，当从常规思路去探索发现是个比较复杂的问题时，我们不妨用一个具体的例子换个“出发点”重新审视一下这个题目，便能找到查找的规律，从而找到解决问题的突破口！（切勿惯性思维）
