题目

给定一个 m x n 二维字符网格 board 和一个字符串单词 word 。如果 word 存在于网格中，返回 true ；否则，返回 false 。

单词必须按照字母顺序，通过相邻的单元格内的字母构成，其中“相邻”单元格是那些水平相邻或垂直相邻的单元格。同一个单元格内的字母不允许被重复使用。

例如，在下面的 3×4 的矩阵中包含单词 “ABCCED”（单词中的字母已标出）。

示例 1：
输入：board = [[“A”,”B”,”C”,”E”],[“S”,”F”,”C”,”S”],[“A”,”D”,”E”,”E”]], word = “ABCCED”
输出：true

示例 2：
输入：board = [[“a”,”b”],[“c”,”d”]], word = “abcd”
输出：false

提示：
m == board.length
n = board[i].length
1 <= m, n <= 6
1 <= word.length <= 15
board 和 word 仅由大小写英文字母组成

题解

思路

“寻找路径”那就是一个搜索问题，首先就是找开头，一下子就写两个for循环嵌套。后面来分析循环内部怎么写？找到开头数在数组中的位置，接着的数只能在其上下左右寻找；对于其四周数判断存在什么情况呢？①数所在位置是边界位置②四周的数都和单词下一个数不等③数字对应相等则继续重复操作寻找单词所在位置。这只是我一开始最浅显的思路。

不过很多具体细节我不知道如何去实现，上面的思路中说到“重复操作”那这便可以用到递归，但是细想递归是一直往深处运行，但是这题似乎有些不同，因为它存在一种情况当一条路走不下去的时候，它要返回上一个点选择路线，但我一开始说不清楚具体不同在上面地方，直到看到解析说这是回溯法。那什么是回溯法呢？

知识点：回溯函数

回溯法也叫做回溯搜索法，它是一种搜索方式。递归返回上一层就是回溯，有递归的地方就有回溯，回溯函数就是递归函数。回溯的本质其实就是一种穷举。其间为了让回溯高效些，会使用一种剪枝的操作，但是改变不了回溯是穷举的本质。（ps：剪枝：顾名思义，就是通过一些判断，砍掉搜索树上不必要的子树。这些子树可能是不可达的，也可能是可达但显然不是最优的，去掉它们对最终答案没有影响，所以我们称为“剪枝”。常用的剪枝分成以下两类：1.可行性剪枝。2.最优性剪枝。）

总结

此题就是典型的矩阵搜索问题，高级说法使用的是深度优先搜索（DFS）+剪枝方法，简单说就是回溯。回看这题，说说有那几个点难住我的，还有就是写完还是一知半解的地方（自我反思）。

难点：①回溯函数的思路理清和书写本身就对我来说难；②对于四个方向上的判断以以及不走回头路；③如果走错了回溯是咋样的进程，我比较难理解

做完之后的自我反思，并且参考了力扣评论区的一些问答:

1.反思一个问题：如果走到一个数它的右边和下边都和单词的数匹配，如果我先走右边发现走不通，我是如何重新走下边的？仔细看下面这段代码，其实它把每个都执行了，只是并不是把选择一条路执行！

2.一个比较难理解的点：递归搜索匹配字符串过程中，需要 board[i] [j] = ‘0/‘ 来防止 ”走回头路“ 。当匹配字符串不成功时，会回溯返回，此时需要board[i] [j] = word[k] 来”取消对此单元格的标记”。 在DFS过程中，每个单元格会多次被访问的， board[i] [j] = ‘0/‘只是要保证在当前匹配方案中不要走回头路，不复原，在下一个匹配方案中会影响最终结果。

3.递归搜索匹配字符串过程中，需要 board[i][j] = '/' 来防止 ”走回头路“ 。当匹配字符串不成功时，会回溯返回，此时需要board[i][j] = tmp 来”取消对此单元格的标记”。 在DFS过程中，每个单元格会多次被访问的， board[i][j] = '/'只是要保证在当前匹配方案中不要走回头路。

4.为什么匹配成功也需要回溯呢，我觉得只有失败了才需要回溯吧？匹配成功也需要将 True 结果一层一层返回，直至起始点，以获取匹配成功的结果。
