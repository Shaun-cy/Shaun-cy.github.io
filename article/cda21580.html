剑指offer19题解

发表于2022-11-04|更新于2022-11-18
|字数总计:1.5k|阅读时长:5分钟

题目

请实现一个函数用来匹配包含’. ‘和’* ‘的正则表达式。模式中的字符’.’ 表示任意一个字符，而 ‘ * ‘表示它前面的字符可以出现任意次（含0次）。在本题中，匹配是指字符串的所有字符匹配整个模式。例如，字符串”aaa”与模式”a.a”和”abaca”匹配，但与”aa.a”和”ab*a”均不匹配。

示例 1:
输入:
s = “aa”
p = “a”
输出: false
解释: “a” 无法匹配 “aa” 整个字符串。

示例 2:
输入:
s = “aa”
p = “a*”
输出: true
解释: 因为 ‘*’ 代表可以匹配零个或多个前面的那一个元素, 在这里前面的元素就是 ‘a’。因此，字符串 “aa” 可被视为 ‘a’ 重复了一次。

示例 3:
输入:
s = “ab”
p = “.“
输出: true
解释: “.“ 表示可匹配零个或多个（’*’）任意字符（’.’）。

示例 4:
输入:
s = “aab”
p = “c * a * b”
输出: true
解释: 因为 ‘*’ 表示零个或多个，这里 ‘c’ 为 0 个, ‘a’ 被重复一次。因此可以匹配字符串 “aab”。

示例 5:
输入:
s = “mississippi”
p = “mis * is * p *.”
输出: false
s 可能为空，且只包含从 a-z 的小写字母。
p 可能为空，且只包含从 a-z 的小写字母以及字符 . 和 ，无连续的 ‘‘。

题解

思路

事先说明：本题思路完全是在看完力扣大佬解析后，将自己对题目的理解以及做题思路的一个简单梳理。

简单分析一下这个题目，两个字符串对比匹配，从左往右依次对边两个字符串中的字符在含有特殊字符作用下，是否匹配。仔细观察它便是一个动态规划的题，不过它是个二维动态规划。

我从代码随想录学到关于写动态规划的五个步骤：

  1. 确定dp数组（dp table）以及下标的含义
  2. 确定递推公式
  3. dp数组如何初始化
  4. 确定遍历顺序
  5. 举例推导dp数组

我觉得比较重要的就是这个递推公式。这边dp[i] [j]表示s中前i-1个元素与p中j-1个元素是否匹配；若 dp[i] [j] == ture：匹配，否则不匹配。这里需要对两种情况进行讨论，就是当p[j-1]是否为’*’的时候进行分情况讨论。

转移方程： 需要注意，由于 dp[0] [0] 代表的是空字符的状态， 因此 dp[i] [j] 对应的添加字符是 s[i - 1] 和 p[j - 1] 。

当 p[j - 1] == ‘*’ 时， dp[i] [j] 在满足以下任一情况时等于 true ：

dp[i] [j - 2] == true： 即将字符组合 p[j - 2] * 看作p[j - 2]出现0次(说简单点就是删除p[j - 2])；例： s：ab p：abc*
dp[i - 1] [j] == true && s[i - 1] == p[j - 2]: 即让字符 p[j - 2] 进行复制；例 s:abb p:ab*
dp[i - 1] [j] == true && p[j - 2] == ‘.’: 即让字符 ‘.’进行复制；**例 s:abb p:a. ***

当 p[j - 1] != ‘*’ 时， dp[i] [j] 在满足以下任一情况时等于true ：

dp[i - 1] [j - 1] == true && s[i - 1] == p[j - 1]：字符串s[i - 1]前的字符和p[i - 1]前的字符匹配且s[i - 1] == p[i - 1]；例 s:abc p:abc
dp[i - 1] [j - 1] == true && p[j - 1] == ‘.’：字符串s[i - 1]前的字符和p[i - 1]前的字符匹配且p[i - 1] == ‘.’；例 s:abc p:ab.

初始化条件

当s和p都为空串时都匹配，即dp[0] [0]=1

(1)判断非空串s和空串p的匹配
显然，这必然不可能匹配，即这一列从i=1开始全为0

2)判断空串s与非空串p的匹配
这种情况只有p带’*’时才有可能匹配

总结

一开始试着做这题的时候，写完代码后提交失败，仔细思考后才发现这道题的难点在哪里。关键就在与’*’的处理，题目描述的是” *表示它前面的字符可以出现任意次（含0次）”，换句话说就是对 * 号前前一个数的删除或复制，难点就在于复制，你很难判断他会复制多少次。并且初次尝试时写的代码用的也是见得的if条件判断，后面看解析才知道要用到动态规划的思想，而且是二维动态规划！！！初次接触一下子让我犯了难，看了好久才勉强理解。

说到动态规划，之前的题目有斐波那契数列，当时我没太在意动态规划的思想，主要它是一维的比较好理解，我把那题的重点放在的哈希表做法的上了，现在看来关于数列，字符串匹配这类重点还是动态规划的思想。

在代码随想录 里好好看了看关于动态规划的知识点和例题，收益颇丰。
