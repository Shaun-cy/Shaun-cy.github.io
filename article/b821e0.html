发表于2022-10-10|更新于2022-10-14
|字数总计:1.1k|阅读时长:3分钟

题目

把一个数组最开始的若干个元素搬到数组的末尾，我们称之为数组的旋转。

给你一个可能存在 重复 元素值的数组 numbers ，它原来是一个升序排列的数组，并按上述情形进行了一次旋转。请返回旋转数组的最小元素。例如，数组 [3,4,5,1,2] 为 [1,2,3,4,5] 的一次旋转，该数组的最小值为 1。

注意，数组 [a[0], a[1], a[2], …, a[n-1]] 旋转一次 的结果为数组 [a[n-1], a[0], a[1], a[2], …, a[n-2]] 。

示例 1：
输入：numbers = [3,4,5,1,2]
输出：1

示例 2：
输入：numbers = [2,2,2,0,1]
输出：0

提示：
n == numbers.length
1 <= n <= 5000
-5000 <= numbers[i] <= 5000
numbers 原来是一个升序排序的数组，并进行了 1 至 n 次旋转

题解

思路

最直观的解法就是遍历数组但是时间复杂度较高。考虑到它是有序数组，于是我想到二分法。不过后面的想法不够缜密，写出的代码不能通过。

先说说我最初的想法吧：旋转之后的数组实际上是分成两段有序数组，且最小值便是分界线。由于它是有序数组，我试着用二分查找法找。首先我用两个指针begin、end分别指向数组第一个元素和最后一个元素，按照题目旋转数组的规则，第一个元素是大于或者等于最后一个元素的（实际不是），接着我用指针mid指向中间位置，这便开始判断区间

1.如果mid指向的数大于begin（例：[3,4,5,1,2]），则前半段数组是递增，因此最小值只会出现在后半段，将begin重新指向mid+1；

2.如果mid指向的数小于begin（例：[4,5,1,2,3]）,则后半段数组是递增，最小值应该在前半段中寻找，将end指向mid；

3.如果mid指向的数等于begin（例：[1,1,1,2,1])，则在后半段！（可是但凡严谨一点就发现如果数组是[1,2,1,1,1]呢，不过但是我没发现，代码写出来出错才发现）

单纯从上面思路的第三步便发现我一开始想法有多么的不严谨，而且还有一点关于旋转数组的规则，在旋转数组中，由于把递增排序数组前面的若干个数字搬到数组的后面，因此第一个数字总是大于或者等于最后一个数字。但是我忽视了一个特例：若干个数也可能是0个，即排序数组本身，这仍然是数组的一个旋转，我们的代码需要支持这种情况。此时，数组中的第一个数字就是最小的数字，可以直接返回。因此应该首先比较如果begin小于end直接输出begin指向的数。

接着我在思考如果相同的情况下该如何判断？《剑指offer》给出的思路是采用顺序查找法；力扣上网友写的答案解析是直接将begin指向begin+1。这两个方法最直观最好理解的是剑指offer书中给出的，但是需要另写一个函数判断比较麻烦。另一个的话需要多想一下，因为mid和begin一样，删掉一个数不影响最小值的寻找。

总结

从理清思路到成功写出代码之间我还是花了不少时间的，主要问题出在while函数的条件和一些特殊情况（比如数组只有一个数）考虑不够周全。

这道题主要就是考察对二分查找的理解吧，特别一些特例的处理要尤为在意！
